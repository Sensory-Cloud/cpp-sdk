SensoryCloud C++ SDK

This repository contains the source code for the SensoryCloud C++ SDK.

General Information

Before getting started you must spin up a SensoryCloud inference server or have SensoryCloud spin one up for you. You must also have the following pieces of information:

  • your inference server address and port number,
  • your SensoryCloud tenant ID, and
  • your configured secret key used for registering OAuth clients.

Requirements

This project uses CMake as the primary build system and supports the gcc, clang, and MSVC compilers. To install the build tools, run the following installation command depending on your deployment target. The minimum requirements for installation of the SDK are:

Dependency Version
cmake >= 3.11
gcc >= 5.1
clang >= 4
MSVC >= 2015

Debian / Ubuntu

To install the build tools for Debian-based Linux:

apt-get install -y build-essential autoconf libtool pkg-config cmake git

Fedora / RedHat / CentOS

To install the build tools for Fedora-based Linux:

yum install -y autoconf libtool pkg-config gcc gcc-c++ cmake git

MacOS

To install the build tools for MacOS:

xcode-select --install
brew install autoconf automake libtool shtool cmake git

Because MacOS is not strictly POSIX compliant, you will need to provide gRPC with a hint of where your SSL certificates are stored to properly connect to a secure server. This can typically be accomplished by exporting the following environment variable:

export GRPC_DEFAULT_SSL_ROOTS_FILE_PATH=/etc/ssl/cert.pem

Integration

To include the SensoryCloud C++ SDK in your project, add the following snippet to your CMakeLists.txt.

include(FetchContent)
FetchContent_Declare(sensorycloud
URL https://codeload.github.com/Sensory-Cloud/cpp-sdk/tar.gz/refs/tags/v1.0.0
)
FetchContent_MakeAvailable(sensorycloud)

This will clone and compile libsensorycloud locally to be used in applications or other libraries using:

target_link_libraries(your_program PRIVATE sensorycloud)

Tutorial & Examples

Please refer to TUTORIAL.md for a brief tutorial of the SDK. Example code for each available service and programming model are available in the [examples](examples) directory. These act both as a point of reference and as a starting point for the integration of SensoryCloud with your application.

Contributing

The CONTRIBUTING.md document describes how to setup the development environment for the SDK, compile the code outside of the context of an application, run unit tests, and release new versions of the SDK. Please refer to the CHANGELOG.md for the latest features and bug fixes.
